# 配置中心与数据结构

## 1. 配置设计目标

- 支持默认配置、远程灰度、用户配置、场景配置、宿主提示配置叠加
- 配置项可热更新
- 配置决策过程可追踪

## 2. 配置优先级

从低到高建议如下：

1. 内置默认配置
2. 词库/策略远程配置
3. 用户设置
4. 当前子类型配置
5. 宿主应用 `extraConfig`
6. 系统安全模式强制规则

最终决策以“更保守的安全策略优先”。

## 3. 全局配置结构

参考样例：

- `design/config/offhand-ime-config.sample.json`

建议分区：

- `app`
- `input`
- `pinyin`
- `rime`
- `english`
- `voice`
- `llmPolish`
- `privacy`
- `sync`
- `experiments`

## 4. 宿主输入框扩展配置

当前 IMEKit 已暴露 `EditorAttribute.extraConfig`，可以承载宿主应用给输入法的提示信息。

建议约定：

```json
{
  "customSettings": {
    "offhand.scene": "chat",
    "offhand.polish.enabled": true,
    "offhand.polish.mode": "light",
    "offhand.voice.enabled": true,
    "offhand.sensitive": false,
    "offhand.domain": "office"
  }
}
```

建议宿主只提供“提示”，最终仍由输入法统一做安全裁决。

## 5. 子类型配置

参考样例：

- `design/config/inputmethod-subtypes.sample.json`

建议每个 subtype 挂以下属性：

- `id`
- `name`
- `locale`
- `language`
- `keyboardLayout`
- `voiceEnabled`
- `llmPolishEnabled`
- `domainHints`
- `schemaId`
- `engine`

## 6. 润色策略配置

参考样例：

- `design/config/llm-polish-policy.sample.json`

建议按“场景 + 模式 + 阈值”配置：

- 哪些场景禁用润色
- 哪些场景默认只给建议不自动替换
- 每种模式的长度限制
- 每种模式的超时和重试
- 是否保留换行、数字、专有名词

## 7. 推荐的客户端数据结构

```ts
export interface InputSessionContext {
  sessionId: string;
  subtypeId: string;
  schemaId?: string;
  textInputType: string;
  enterKeyType: string;
  securityMode: 'BASIC' | 'FULL';
  bundleName?: string;
  abilityName?: string;
  placeholder?: string;
  extraConfig?: Record<string, number | string | boolean>;
}

export interface CandidateItem {
  id: string;
  text: string;
  source: 'user' | 'core' | 'domain' | 'hot' | 'voice' | 'llm';
  score: number;
  annotation?: string;
}

export interface PolishSuggestion {
  originalText: string;
  polishedText: string;
  mode: 'light' | 'formal' | 'clear' | 'friendly';
  latencyMs: number;
  diffRanges?: Array<{ start: number; end: number }>;
}
```

## 8. 配置仓库设计

`ConfigRepository` 建议包含：

- `loadDefaults()`
- `loadRemoteFlags()`
- `loadUserSettings()`
- `mergeForSession(ctx)`
- `explainDecision(key)`
- `buildRimeRuntimeConfig(ctx)`

其中 `explainDecision(key)` 用于开发调试，回答“为什么这个输入框不允许润色”。

## 9. 配置持久化

建议本地分层存储：

| 数据 | 存储建议 |
| --- | --- |
| 用户开关项 | Preferences |
| 用户词库 | 数据库或结构化文件 |
| 热更新版本信息 | Preferences |
| 远程灰度缓存 | 本地 JSON |
| LLM 会话临时态 | 内存 |

## 10. 灰度与实验

实验开关建议至少包含：

- `enable_voice_input`
- `enable_llm_polish`
- `enable_auto_polish_after_voice`
- `enable_fuzzy_pinyin`
- `enable_domain_lexicon`
- `enable_preview_text`

注意：

- 实验开关不能突破安全模式
- 用户手动关闭某能力时，灰度不能强行重新打开
