# 隐私安全运维与测试

## 1. 隐私原则

输入法天然高敏感，设计必须默认保守。

原则：

- 默认不开启云能力，需用户显式授权
- 默认不上传任何密码、验证码、链接、邮箱、数字账号
- 仅上传完成润色所需的最小文本
- 不记录原始键盘逐键日志
- 所有云请求必须可关闭、可撤回、可审计
- 用户可一键导出/删除个人数据（词库、历史润色请求的审计摘要）

合规对齐：

- 中国《个人信息保护法》：最小必要、明示同意、用户可撤回
- 《生成式人工智能服务管理暂行办法》：训练/输入数据可追溯，具备内容安全过滤
- HarmonyOS 应用市场审核：麦克风与网络权限必须有显式理由，首次调用需动态申请

## 2. 数据分级

| 数据类型 | 级别 | 处理规则 |
| --- | --- | --- |
| 密码/验证码 | 极高 | 不存、不学、不传 |
| 普通输入文本 | 高 | 本地优先，云端按需 |
| 用户词频 | 高 | 本地存储，可选加密同步 |
| 热词包 | 中 | 可下载缓存 |
| 诊断指标 | 中 | 脱敏上报 |

## 3. LLM 请求安全策略

- TLS 传输
- 鉴权 token 短期有效
- 超时默认 `1200ms`
- 服务端不保留原文或只保留脱敏审计摘要
- 企业版支持私有网关

## 4. 安全拦截规则

以下情况一律禁止语音与润色：

- `SecurityMode.FULL`
- `VISIBLE_PASSWORD`
- `NUMBER_PASSWORD`
- `SCREEN_LOCK_PASSWORD`
- `ONE_TIME_CODE`
- 宿主通过 `extraConfig` 标记 `offhand.sensitive=true`

以下情况默认禁止润色：

- `EMAIL_ADDRESS`
- `URL`
- `PHONE`
- `NUMBER`

## 5. 稳定性策略

### 5.1 超时

- 语音引擎初始化超时：`800ms`
- 语音最终结果等待：`1500ms`
- LLM 润色超时：`1200ms`

### 5.2 重试

- LLM 请求最多一次快速重试
- 热词拉取使用指数退避
- 词库下载失败不影响基础输入

### 5.3 熔断

当服务端在 5 分钟内连续失败达到阈值时：

- 自动关闭自动润色建议
- 保留手动触发入口
- 页面静默提示“智能能力暂不可用”

## 6. 可观测性

### 6.1 技术栈

| 用途 | 载体 | 说明 |
| --- | --- | --- |
| 业务埋点 | `@ohos.hiviewdfx.hiAppEvent` | 子类型切换、候选点击位置、润色接受率等 |
| 运行日志 | `@ohos.hiviewdfx.hilog` | 分 domain `0xD003900` 起步，tag `offhand_ime` |
| 系统异常 | `@ohos.hiviewdfx.hiSysEvent` | 监听键盘崩溃、ANR、内存水位 |
| 性能剖面 | `@ohos.hiviewdfx.hiTraceMeter` | 按键 → 候选刷新 trace，首帧 trace |

### 6.2 核心指标

- 子类型切换次数
- 候选点击位置分布
- 语音成功率与首部分/最终结果耗时
- LLM 请求量、成功率、超时率、熔断次数
- 脱敏命中分类与拒绝率（不记录原始片段）
- 润色建议接受率与回退原文比例
- 端侧润色可用率（P2）

### 6.3 日志规则

- 使用 `requestId`（uuid v4）串联 ASR → 脱敏 → LLM → UI 展示全链路
- 只记录文本长度、语言、子类型，不记录文本内容
- 开发模式（`debuggable=true`）下可记录脱敏后样本，默认 7 天滚动清理
- 严格分离 `HiLog.INFO` 与用户敏感路径：任何路径上禁止 `HiLog.info(..., rawText)`

### 6.4 远程 kill switch

- 配置中心下发 `feature_flags` 带版本号与 TTL
- 客户端每 5 分钟拉一次；收到 `disable_llm_polish=true` 立即生效
- 熔断计数（`5 分钟内失败 ≥ 20 次`）触发本地自动 kill，与远程开关合并取更保守值

## 7. 测试策略

### 7.1 单元测试

- 拼音切分
- 候选排序
- 中英混输识别
- 配置优先级合并
- 安全模式决策

### 7.2 集成测试

- `inputStart/inputStop` 生命周期
- `insertText/deleteBackward/setPreviewText`
- 语音识别会话完整流程
- LLM 超时与回退

### 7.3 数据测试

- 词库去重
- 热词污染检查
- Blocklist 生效
- 用户学习衰减逻辑

### 7.4 UI 测试

- 竖屏/横屏
- 小屏/大屏
- 固定键盘/浮动键盘
- 中英切换动画与焦点状态
- 深浅色主题、跟随系统色
- 无障碍：
  - 按键可被 `Accessibility` 焦点命中，朗读键名
  - 候选栏元素提供 `accessibilityText`
  - 震动/音效可关闭，默认跟随系统触感设置
  - 动态字号跟随系统设置，最大不突破键盘高度 1.3 倍

### 7.5 安全测试

- 密码框拦截
- FULL 安全模式拦截
- 崩溃日志不带文本原文
- 网络抓包验证最小上传字段

## 8. 验收清单

### MVP 验收

- 中文拼音输入稳定
- 英文输入稳定
- 可切换子类型
- 用户词频学习生效
- 手动润色可返回建议
- 密码/验证码场景全部拦截云能力

### 智能增强验收

- 语音输入可用且可中断
- 语音结束后能展示原文和润色建议
- LLM 超时不影响继续输入
- 行业词库可热更新

## 9. 运维建议

- 词库与 LLM 策略分开发布
- 先灰度 1%，再扩大到 10%、30%、100%
- 建立“输入法性能看板”和“智能能力质量看板”
- 为高风险配置保留远程 kill switch
